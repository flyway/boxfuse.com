Boxfuse is able to automatically provision both PostgreSQL and MySQL databases locally and on AWS RDS. Boxfuse exposes the database URL, user and password as environment variables within the application instances. And if your application is based on Spring Boot, Grails, Play or Dropwizard, Boxfuse will automatically configure your frameworks's default datasource with those settings.

This is a great fit for microservices that follow the database per service:

dev environment

Locally on your machine, for your dev environment Boxfuse installs and launches a dev VM that contains a fully-configured PostgreSQL database. The PostgreSQL service from that VM is then exposed to your app by mapping its port.

test and prod environments

On AWS, for your test and prod environment Boxfuse automatically provisions either single- or multi-AZ RDS instances for your application.

To ensure your infrastructure can survive in the face of failure, Boxfuse can make your RDS database highly available by provisioning a continuously replicated geo-redundant hot standby spare instance that's ready take over in case the primary one fails.

Security

The user provided has superuser privileges.

All database connection use SSL (TLS). Every Boxfuse instances comes with RDS root certificates out of the box.

Auto-configuration

If your application is based on Spring Boot, Grails, Play or Dropwizard, Boxfuse will automatically configure your framework's default datasource with the correct database driver class name, URL, user and password.

Schema evolution / database migration

Evolving your schema works best with Flyway.

We strongly recommend you apply datbase migrations on application startup. You can either use a plugin for your framework of choice or call Flyway programmatically:

Flyway flyway = new Flyway();
flyway.migrate();

Flyway 4.0 and newer comes with out of the box support for Boxfuse. When running inside a Boxfuse instance Flyway will automatically pick up the database URL and credentials exposed by Boxfuse.

Time Zones

All databases created by Boxfuse are created with their time zone set to UTC.

Manual connections

You can manually connect to the correct database for each environment by using the open -db command. This will automatically create a secure connection using your database's native client:

> boxfuse open -db -env=test
