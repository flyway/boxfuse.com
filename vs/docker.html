Docker vs Boxfuse

Boxfuse is an integrated, fast and much easier alternative to Docker for running JVM and Node.js apps on AWS.
Here's why you should consider switching...

Thinking of making the switch to Boxfuse to run your JVM and Node.js apps on AWS?

We want to make the decision as easy as possible, so we’ve put together some info that’ll help you decide which option is best for you.

Below, you'll find information to help you compare Boxfuse and Docker, including features, pricing and thoughts from current Boxfuse customers who have made the switch.

1. Boxfuse is much easier and better integrated than Docker

Boxfuse is all about making you productive so you can focus on delighting your customers instead dealing with infrastructure issues. We made sure Boxfuse is a joy to use by making running JVM and Node.js applications on AWS absolutely trivial. There are no lengthy tool chains to master, no complicated integrations and no tedious OS provisioning and hardening. Instead all you need is a single command, boxfuse run, which will take you all the way from JVM or Node.js application to fully-provisioned AWS infrastructure with no configuration required:

Boxfuse does this by first analysing your application and then that information to automatically generate an image with a minimal Linux-based OS on-the-fly. Boxfuse then uploads your new image to AWS, converts it to an AMI and provisions all the necessary infrastructure in the region of your choice.

Boxfuse runs natively on Windows, Mac and Linux and comes with special optimizations for popular tools and frameworks like Spring Boot, Dropwizard, Grails, Play, Executable Jars, Tomcat, TomEE and Node.js.

Boxfuse goes well beyond what you would expect and is able to read your framework's native configuration files and automatically configures security groups, load balancers and more based on the port and healthcheck information they contain, with no additional effort required on your part (needless to say, you are of course able to override these defaults should you want to).

3. Boxfuse is built for Continuous Deployment

Continuous Deployment is critically important for many companies today, allowing them to innovate and iterate faster.

Boxfuse fully embraces this by treating build tool plugins as first-class citizens, making it effortless to integrate AMI building into your pipeline. In addition to its command-line client, Boxfuse comes with both Maven and Gradle plugins which can be readily integrated through the usual means:

Gradle: build.gradle

plugins {
    id "com.boxfuse.client" version "{{site.boxfuseVersion}}"
}

Maven: pom.xml

<plugin>
    <groupId>com.boxfuse.client</groupId>
    <artifactId>boxfuse-maven-plugin</artifactId>
    <version>{{site.boxfuseVersion}}</version>
</plugin>

For rapid turnarounds and fast feedback, Boxfuse also lets you run your images on VirtualBox. This is a great way to run your integration tests locally before creating an AMI. And unlike Docker, Boxfuse's client runs natively on Windows, Mac and Linux with no special shell or setup required.
